Hello, thanks for submitting an issue report and helping make Angular better.

Here are some guidelines to help you create an issue reproduction that developers will love you for:
  • Minimal reproduction. Only code, markup and styles absolutely required to reproduce the issue.
  • Single component with inlined markup. In most cases, a single component with an inline template is all that is required to reproduce an issue.
  • Step-by-step instructions For example:
    • Step 1: Fill out textbox
    • Step 2: Select value x from dropdown
    • Step 3: Click the "Show me" button, This produces the following error "....".
  • Expected and Actual behavior.
    • Expected: View updated with user input
    • Actual: Error message in console, view not updated
  • Example reproduction
